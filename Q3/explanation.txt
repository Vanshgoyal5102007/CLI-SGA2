Explanation of System Calls
open(): This call accesses the file at the operating system level, using flags to create it if it doesn't exist and open it for both reading and writing.
write() and read(): These system calls handle raw, unbuffered data transfer directly between the program's memory structures and the file descriptor.
lseek(): This allows efficient updates by moving the file offset pointer directly to a specific byte location (calculated via sizeof) so a single record can be overwritten without rewriting the entire file.
close(): This safely terminates the connection to the file and releases the descriptor back to the Linux kernel.
