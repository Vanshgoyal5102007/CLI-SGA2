Explanation of Concepts
Process Creation (fork): I used fork() to clone the parent process, allowing the child to handle tasks independently just like a web server worker process.
Termination (kill): When the child became unresponsive, the parent used the kill() system call to send a SIGKILL signal, forcing the child to shut down immediately.
Waiting (waitpid): By calling waitpid(), the parent process paused to collect the child's exit status. This step is mandatory because it removes the terminated child from the system process table, successfully preventing a zombie process.
