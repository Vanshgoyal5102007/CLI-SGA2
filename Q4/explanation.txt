Pipeline Design Justification
tail -f: This command keeps the file descriptor open and reads new data in real-time as the file grows.
2>/dev/null: This redirection suppresses unnecessary standard error messages by sending them to the null device, ensuring our pipeline only processes actual log data.
grep --line-buffered: This extracts only the lines matching our error criteria. The line-buffered flag forces it to pass the matched data down the pipeline immediately instead of holding it in memory.
tee -a: This command duplicates the data stream, displaying the error on the standard output for the administrator while simultaneously appending it to the report file.
